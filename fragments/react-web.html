So what is React?

We already know what it's not, so let's take a look at what it is...

We'll focus on React for the web

React is...

A better way to think of your frontend.

React allows you to think of `UI = f(data)`. This is important because programming UIs is hard. You have to reason about user actions and change your UI's state accordingly - and track that state.

React is also fast!

Web UI programming

When reasoning about your front-end, you're considering the structural markup required to create your UI, the CSS styles that define how the UI looks, and the Javascript that modifies the DOM in order to update your UI's state.

Typically you reason this out by separating your concerns and maintaining modular code. And possibly some build system that uses Node.js. This will get very complex the bigger your project and the snazzier you make your UI.

Typical web project

/project
    /app
        index.html
    /partials
        header.hbs
        footer.hbs
        block.hbs
        section.hbs
    /css
        main.scss
        _footer.scss
        _header.scss
    /js
        /lib
        app.js

Web UI programming

React helps you manage this complexity by relaxing this separation of concerns and instead focus on creating and composing components.

We'll discuss components in a bit...

Web UI programming

A lot of the heavy lifting is done by CSS, but when you want to make a very interactive UI, you need to toggle styles or show/hide elements. This is where jQuery shines with easy DOM manipulation. 

<div class="tumble">Some text.</div>
$( "div.tumble" ).toggleClass( "bounce" )

The DOM

Manipulating the DOM is expensive. Each time you toggle styles or visiblity, the DOM needs to re-render its layout. This is why you sometimes see janky website animations

Most frameworks that modify the DOM have to worry about layout thrashing

React Virtual DOM

React React introduces the shadow DOM or virtual DOM. This prevents layout thrashing by queing DOM updates in a 'shadow' copy of the DOM. React then figures out when's the best time to copy its changes to the real DOM.

Think of this as a diff. React looks at your current layout, and if there's an update in the state, then it'll redraw the layout.
