CCLE

UCLA's main course management system.

Problem

The system is built on Moodle, a monolithic legacy application (read: ugly system)

UI is looking old and tired. Stakeholders are demanding a more modern and mobile UI, but this is not a priority for upstream maintainers.

The underlying theme system is incredibly complex.

Changes to the UI are done by committee. Known to be a drawn out and traumatic process for participants.

Proposal

Make UI consistent. Make all UI elements share similar theme. Use consistent color scheme. Use consistent fonts. Usability and accesibility focus.

Integrate modern CSS framework (like Boostrap)

Make it mobile friendly. Needs to be good looking and functional.

Overpromise...

Rewriting the theme system is not an option. It's difficult and time consuming, and would need to convince Moodle community.

Need alternate approach...

SASS + Compass

Build a custom CSS toolchain using SASS + Compass (Ruby gems)

CSS build system is incredibily flexible and allows importing any external CSS framework as well as overriding current styles.

Possible to inject Boostrap framework styles into existing theme classes using SASS `@extend` construct.

No need to touch underlying Moodle theme system. My system sits on top and uses the CSS cascade to override the theme.

Resposive

Achieved responsive design on a legacy system through clever user of breakpoints.

Legacy applicaction becomes mobile friendly overnight.

https://github.com/ucla/moodle/blob/master/theme/uclashared/sass/ucla/_responsive.scss

Overdeliver

Build system works better than expected. Allows us to upgrade from Boostrap 2 to Bootstrap 3 in a single sprint

Campus reports general rise in happiness

Presented my methods at Moodle conference and Moodle community takes note, but they've already invested in rewriting their theme to use Boostrap styles. Having intially targetted Boostrap 2, upgrading from Boostrap 2 to Boostrap 3 takes them 6 months!

My CSS build system is still in place
