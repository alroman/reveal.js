More about components

Big pile of components

Your whole react app is just one big pile of components. And each component is built of even more components under the hood. 


    <App>
        <Intro>Hello World</Intro>
        <Content/>
        <Footer links={this.state.links} />
    </App>

Be the component

You already work with components. HTML is full of them and your browser already knows how to render them and how to handle user interaction. 


<select name="select">
  <option value="value1">Value 1</option> 
  <option value="value2" selected>Value 2</option>
  <option value="value3">Value 3</option>
</select>

Be the component

Think of <App> as just another component, but in this case React tells the browser how to render and how to, uh..., react with it

State & props

Each component maintains a state and can potentially have props (properties).

React data flow is `unidirectional`. Data can only go from parent components to their children.

Props

Similar to the HTML element properties that you already know.

Properties are instantiated when a component is created.

Properties are how components talk to each other and can be used to set up the state of a child component.

State

State is mostly a simple object.

State is managed by the owning component and it's never shared, however it can be passed down to its own children as a prop. 


        constructor(props) {
            super(props)
            this.state = {
                color: 'green',
                list: [
                    'first',
                    'second',
                ]
            }
        }

State...

You never modify state directly. You modify the state by calling seState(obj) 


        this.setState({color: 'blue'})

State updates are async, and react decides how to 'merge' updates.
