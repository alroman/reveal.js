Mobilize Labs

Current position as mobile + web developer. Showcasing our app content management platform (work in progress).

We produce a lot of mobile apps and we want to make app deployment easier for ourselves. Goal of CAP is to provide a dynamic way for mobile app owners to update their app content with reusable templates. In theory this will allow us to build mobile apps quicker, will allow users to share templates, and eventually reduce our support costs.

CAP

The CAP system is composed of three separate architectural components

  • Authentication system
  • API Server
  • Web frontend

Architecture Overview

Devops

AWS deployment with S3 bucket to serve front-end. Elastic Beanstalk + Node for Express server. EC2 Ubuntu instance running CouchDB. EC2 intance running Keycloak instance.

Loosely coupled architecture allows us to change any single system in the future.

Authentication System

Using Keycloak in order to provide secure SSO and Oauth/JWT authentication.

We also provide Authentication brokering into the campus SSO. This is notable because it allowed me to leverage my IDM skills, and we are also the only group within UC able to do this.

API Server

Using Node with Express server. Leveraging middleware architecture to build RESTful API routes.

Server provides endpoints for the web frontend which allows content owners to edit their content. It also publishes endpoints that CAP apps consume.

Uses JWT for authenticating secure API requests.

Web Frontend

Built with React.js + Redux. Allows user to modify the state of their mobile apps in a dynamic way.

React Native Apps

The end goal is to drive native mobile apps. We use React Native to build the apps for iOS and Android. These apps retrieve their content from CAP.
