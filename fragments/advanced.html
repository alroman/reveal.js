State management

How you manage react apps is by managing component state. The architecture pattern defined by Facebook is called 'Flux' which reiterates the idea that `UI = f(data)`

By managing state, you manage the complexity of your app.

Flux

"Flux is a pattern for managing data flow in your application. The most important concept is that data flows in one direction. "

https://facebook.github.io/flux/

Redux

Redux is an opinionated implementation of this pattern. "Redux is a predictable state container for JavaScript apps"

Redux introduces actions and reducers. Actions are explicit and predictable. Does not necesseraly require React.

MobX

Another take on state management that does not impose the stricness of Redux. It simplifies state management with 3 steps (but also introduces `@` decorators):

  1. Define your state and make it observable
  2. Create a view that responds to changes in the State
  3. Modify the State

https://mobx.js.org/intro/overview.html

Realm

Realm is an object store database. "Realm JavaScript enables you to efficiently write your app’s model layer in a safe, persisted and fast way. It’s designed to work with React Native and Node.js."

https://realm.io/docs/javascript/latest/

**disclaimer: I love realm. It's incredibly easy to use.

Realm...


const Realm = require('realm');

class Person {}
Person.schema = {
    name: 'Person',
    primaryKey: 'name',
    properties: {
        name: 'string',
        age: {type: 'int', default: 0},
    },
};

const realm = new Realm({schema: [Person]});

// Query
let people = realm.objects('Person', 'age >= 17');
people.length // => 0

// Write
realm.write(() => {
    savedPerson = realm.create('Person', {
        name: 'Hal Incandenza',
        age: 17,
    });
});

// Queries are updated in real-time
people.length // => 1
